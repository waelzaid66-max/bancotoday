# تقرير 05 — إخفاقات الفصل والترتيب (FI / Banks / Seller)

**الغرض:** توثيق أماكن “الذوبان” بين عالم البنك وعالم البائع/التاجر — بدون إصلاح.

---

## 1) خريطة العوالم المفروض تكون منفصلة

```
[مشتري] → طلب تمويل على إعلان
                ↓
[BANCO Admin] → مراجعة مخاطر → Forward لوسيط
                ↓
[مؤسسة مالية] → Inbox / فرع / موظف → تواصل / إقفال

بالتوازي:
[حساب FI] → توثيق KYC → ربط owner → تشغيل inbox
```

أي تداخل مع:
- إنشاء إعلانات بيع  
- دور dealer  
- اشتراكات الشركات  

يُعتبر فشل فصل.

## 2) نقاط الذوبان المثبتة

### S1 — CTA البنوك → onboarding بدون نوع حساب
- **أين:** `app/business/banks.tsx` join CTA  
- **الأثر:** فرد → غالباً `dealer` رغم نشاط بنك  
- **الشدة:** عالية

### S2 — فورم onboarding مشترك
- **أين:** `app/business/onboarding.tsx`  
- **الأثر:** بنك وشركة ومصنع في نفس الحقول ونفس شاشة النجاح  
- **الشدة:** متوسطة–عالية (ترتيب منتج)

### S3 — نجاح onboarding → إنشاء إعلان
- **أين:** CTA `router.replace("/listings/create")`  
- **الأثر:** رحلة بنك تُدفع لمسار بائع  
- **الشدة:** عالية للفصل المفاهيمي

### S4 — البائع العادي ينشر `bank_finance`
- **أين:** `listings/create.tsx`  
- **ملاحظة:** قد يكون مقصوداً (خيارات تمويل على الإعلان)  
- **لكن:** يختلط مع “المؤسسة المالية” كدور منفصل في ذهن المستخدم  
- **الشدة:** متوسطة (يحتاج قرار منتج صريح)

### S5 — اشتراكات الأعمال تستبعد FI
- **أين:** `SubscriptionService` `BUSINESS_ROLES` بدون `financial_institution`  
- **وخطة** `bank_featured` audience = company  
- **الأثر:** لا مسار اشتراك/باقة واضحة للبنك  
- **الشدة:** متوسطة

### S6 — هب عام ثابت + Inbox خاص
- **أين:** نفس صفحة `/business/banks`  
- **الأثر:** الزائر يظن القسم “بيانات بنوك”؛ الحقيقة inbox مخفي لأعضاء فقط  
- **الشدة:** متوسطة (توقعات)

### S7 — أدمن: API Phase 2 بدون UI
- **أين:** `admin-os` vs OpenAPI  
- **الأثر:** التشغيل اليومي لا يقدر يكمل الفصل (ربط بنك ↔ وسيط)  
- **الشدة:** عالية تشغيلية

### S8 — توثيق أعمال عام لكل الأدوار
- **أين:** `is_verified` عام + لا مراجعة مستندات  
- **الأثر:** موافقة “بنك” لا تختلف عن موافقة تاجر  
- **الشدة:** عالية امتثال

## 3) ما هو مفصول بشكل صحيح

| عنصر | لماذا ينجح |
|------|------------|
| `user_role = financial_institution` | enum مستقل |
| Inbox عضوية intermediary/seat | مش dealer leads |
| صلاحية `manage_financing` | منفصلة في الأدمن |
| لون/هوية القسم الأزرق للبنوك | ذاكرة `banco-section-identity` |
| Agent لا يوجّه فروع | إنفاذ دور في الخدمة |

---

## 4) ترتيب موصى به للإصلاح (عند التفويض فقط)

1. فصل مسار التسجيل: Banks CTA يضبط `account_type` أولاً  
2. فورم FI أو على الأقل شاشة نجاح/حقول مختلفة عن البائع  
3. Admin: ربط owner + فروع + مقاعد قبل أي إطلاق تشغيلي  
4. Admin: مراجعة مستندات قبل Verify  
5. تشديد inbox status + تأكيدات النقل  
6. قرار منتج: هل البائع ينشر `bank_finance` أم المؤسسات فقط؟
