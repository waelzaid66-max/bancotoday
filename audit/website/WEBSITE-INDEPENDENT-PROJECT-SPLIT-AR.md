# فصل مشروع الويب عن الموبايل — سجل قرار

**التاريخ:** 2026-07-19  
**القرار:** الويب سايت = مشروع مستقل جديد. الموبايل يبقى كما هو (أقسام منفصلة).

## لماذا
تعليمات المالك تكررت بوضوح: لا تُبنَى تجربة الويب كـ«صورة كاملة» من تطبيق الموبايل
داخل نفس أثر الموبايل. المطلوب كوبي بعيد / مشروع جديد، مع إبقاء الموبايل منفصلًا.

## ماذا تم
| العنصر | الحالة |
|--------|--------|
| `artifacts/banco-website` | مشروع Next.js مستقل (نسخة كاملة من عمل الويب السابق) |
| `artifacts/banco-web` | **مجمّد** (`FROZEN.md`) — لا ميزات جديدة |
| `artifacts/banco-mobile` | منفصل — إصلاح فصل الأقسام في PR مستقل |

## قواعد بعدها
1. أي feature ويب → `banco-website` فقط.
2. أي feature موبايل → `banco-mobile` فقط.
3. لا تعديل API / dealer-os / admin-os من أجل الويب إلا بطلب صريح.
4. Cutover نشر لاحقًا: تحويل Docker/Compose من `banco-web` → `banco-website`.
