# مبادئ تشطيب البرودكشن — حكم التنفيذ من Cursor

**تاريخ التثبيت:** 2026-07-19  
**مصدر:** توجيه المالك — نظام بملايين المستخدمين · ممنوع التجريب · ممنوع تكبير المشاكل.

---

## 1) من يحكم التنفيذ؟

| طرف | الدور |
|------|--------|
| **Cursor (هنا)** | أعلى سلطة تنفيذ للمنتج/الكود: جرد · تصليح · حراس · مراجعة · دمج |
| **Replit** | منفّذ تشغيل فقط: SHA · Expo · شوتات · لوجات — **لا كود** |
| **Copilot** | **UNTRUSTED** — لا اعتماد |

> أي تاسك يمس ملايين المستخدمين = Cursor فقط، بدقة قصوى ومراجعة واختبار.

---

## 2) قواعد حديدية (لا تُكسَر أثناء التشطيب)

1. **NO-WIPE** — لا حذف بوابات / أقسام / Stay / Banks / Legal / FAB / خريطة «للتنظيف».
2. **عزل website** — `artifacts/banco-website` خارج موجات الموبايل.
3. **لا W3 FI** حتى Start صريح بعد دمج #28.
4. **لا بيانات ملفّقة** — لا مخزون وهمي يُعرض كحيّ؛ honesty copy حيث لا يوجد شركاء أحياء.
5. **لا ذوبان Discover→Search** — كل قسم = ميني-آب `/section/*` بـ `useSearchMiniApp` خاص + `lockCategory`.
6. **لا تجريب واسع** — إصلاح مثبت بدليل ملف:سطر فقط؛ إن لم يُثبت عيب = لا تلمسه.
7. **طبقات الحقيقة** — CI أخضر ≠ جهاز ≠ API منشور ≠ متجر. لا تعلن GO من طبقة واحدة.
8. **تكبير المشاكل محظور** — لا refactors «للجمال» · لا توحيد يهدم عزل الأقسام · لا لمس مسار النشر.

---

## 3) سلم التشطيب (ترتيب إلزامي)

```text
[1] جرد صادق (كود+حراس) → وثّق PASS/GAP/RISK
[2] إصلاح P0/P1 مثبت فقط → حارس يمنع الرجوع
[3] اختبارات node الرسمية للموبايل
[4] إثبات جهاز/Replit (شوتات فقط) على SHA محدد
[5] دمج #37 ثم #28 بعد smoke
[6] OPS (أسرار/EAS) — مالك فقط
```
